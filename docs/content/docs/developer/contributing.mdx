# Contributing to Onlook

We welcome contributions to Onlook! This guide will help you get started with contributing to the project.

## Getting Started

Before contributing, please:

1. Read our [Code of Conduct](https://github.com/onlook-dev/onlook/blob/main/CODE_OF_CONDUCT.md)
2. Set up the project locally following the [Running Locally](/docs/developer/running-locally) guide
3. Familiarize yourself with the [Architecture](/docs/developer/architecture) and [Core Systems](/docs/developer/core-systems)

## Contribution Process

1. **Find an Issue**: Look for open issues on our [GitHub Issues](https://github.com/onlook-dev/onlook/issues) page or create a new one
2. **Discuss**: For significant changes, please discuss your approach in the issue before starting work
3. **Fork and Clone**: Fork the repository and clone it locally
4. **Create a Branch**: Create a branch for your changes
5. **Make Changes**: Implement your changes following our coding standards
6. **Test**: Test your changes thoroughly
7. **Submit a Pull Request**: Submit a pull request with a clear description of your changes

## Pull Request Guidelines

When submitting a pull request:

1. Ensure your code follows our coding standards
2. Include tests for your changes
3. Update documentation as needed
4. Write a clear description of your changes
5. Link to any related issues

## Development Workflow

1. **Install Dependencies**: Run `bun install` to install dependencies
2. **Run Development Server**: Run `bun dev` to start the development server
3. **Test Changes**: Test your changes thoroughly before submitting a pull request

## Coding Standards

- Follow the existing code style
- Write clear, concise, and descriptive code
- Include comments where necessary
- Write tests for your code

## Getting Help

If you need help with contributing, please:

1. Check the documentation
2. Ask questions in the issue you're working on
3. Join our [Discord community](https://discord.gg/onlook)
