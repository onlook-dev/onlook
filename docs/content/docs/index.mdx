# Running Onlook Locally

This guide will set you up with running Onlook locally. This involves a Next.js app, a Supabase backend, and the ability to run the dev containers with Codesandbox.

## Prerequisites

These are the prerequisites for running Onlook locally.

- [Bun](https://bun.sh) - Enables running the monorepo
- [Docker](https://docs.docker.com/get-started/get-docker/) - Enables running the Supabase backend

## Setup

### 1. Clone the repository

   ```bash
   git clone https://github.com/onlook-dev/onlook.git
   cd onlook
   ```

### 2. Install dependencies

   ```bash
   bun install
   ```


### 3. Get API keys

#### a. Get Codesandbox API key

    Request an API key from Codesandbox. This is used to run the dev containers. It should be free to use: https://codesandbox.io/api

    Note: We plan on enabling running the dev containers with Docker Desktop in the future after we figure out the best api. 


#### b. Get Anthropic API key (optional)

    Request an API key from Anthropic. This is used for chatting with your project. https://console.anthropic.com/settings/keys


### 4. Run backend

   Make sure you have Docker running from the instructions above.


   ```bash
   bun backend:start
   ```

    Grab the `anon key` from the output. We'll use these in the next step.

### 5. Set environment variables

   Create a copy of the `.env.example` file under the `apps/web/client` directory and name it `.env`. Fill in the values with your own API keys.

    It should look like this:

    ```properties
    # Supabase
    NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321
    NEXT_PUBLIC_SUPABASE_ANON_KEY=<Fill in from content after running supabase start>

    # Codesandbox
    CSB_API_KEY=<Your api key from https://codesandbox.io/api >

    # Drizzle
    SUPABASE_DATABASE_URL=postgresql://postgres:postgres@127.0.0.1:54322/postgres

    # Anthropic
    ANTHROPIC_API_KEY=<Your api key from https://console.anthropic.com/settings/keys >
    ```

### 6. Initialize the database

Set up the database schema
   ```bash
   bun db:push
   ```

### 7. Seed the database with test data

Set environment variables for the seed script under the `packages/seed` directory.

    ```properties
    SUPABASE_DATABASE_URL=postgresql://postgres:postgres@127.0.0.1:54322/postgres
    SUPABASE_URL=http://localhost:54321
    SUPABASE_SERVICE_ROLE_KEY=<Your service role key from Supabase>
    ```

Seed the database with test data. This will create the user along with a test project.
   ```bash
   bun seed
   ```

### 8. Run development server

   ```bash
   bun dev
   ```
